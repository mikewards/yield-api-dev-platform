1

Create Your Account

Start by creating a TBD developer account. You'll need a username and password.

cURL 
curl https://api.tbd.com/v1/accounts \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{
    "username": "your_username",
    "password": "your_secure_password",
    "email": "your_email@example.com"
  }'
Response: 
{
  "account_id": "acc_1234567890abcdef",
  "username": "your_username",
  "created_at": "2025-01-15T10:30:00Z",
  "status": "active"
}

Note: You can also create an account through our web interface.

2

Generate a Personal Access Token

Authenticate with your account credentials to receive an access token, then create a Personal Access Token for API calls.

Step 2a: Authenticate

cURL 
curl https://api.tbd.com/v1/auth/authenticate \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{
    "username": "your_username",
    "password": "your_secure_password"
  }'
Response: 
{
  "access_token": "sk_live_temp_...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "account_id": "acc_1234567890abcdef"
}

Step 2b: Create Personal Access Token

cURL 
curl https://api.tbd.com/v1/access-tokens \
  -X POST \
  -H "Authorization: Bearer sk_live_temp_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My API Key",
    "expires_in": 31536000
  }'
Response: 
{
  "token_id": "tok_1234567890abcdef",
  "access_token": "sk_live_1234567890abcdef",
  "name": "My API Key",
  "created_at": "2025-01-15T10:30:00Z",
  "expires_at": null
}
⚠️ Important: Save your access token securely. You won't be able to see it again after creation. Use this token in the Authorization: Bearer header for all API requests.
3

Make Your First API Call

Now that you have an access token, let's create your first yield account and start earning 6% yield.

cURL 
curl https://api.tbd.com/v1/yield/accounts \
  -X POST \
  -H "Authorization: Bearer sk_live_1234567890abcdef" \
  -H "Content-Type: application/json" \
  -d '{
    "currency": "USDC",
    "initial_deposit": {
      "amount": "1000.00",
      "currency": "USDC"
    },
    "protocol_preference": "auto"
  }'
Response: 
{
  "account_id": "ya_1234567890abcdef",
  "currency": "USDC",
  "protocol": "morpho",
  "annual_yield_rate": 0.06,
  "status": "active",
  "created_at": "2025-01-15T10:30:00Z",
  "balance": {
    "amount": "1000.00",
    "currency": "USDC"
  }
}

🎉 Success! You've created your first yield account. Your funds are now earning 6% annual yield through TBD's integration with Morpho and Aave protocols.

Core Concepts

Accounts & Authentication

Your TBD account is your identity on the platform. Each account can create multiple applications and generate API keys for authentication.

Applications & API Keys

Applications organize your projects and automatically get Ethereum wallets. Each application can have multiple API keys scoped to sandbox or production environments.

Wallets

Each application automatically receives a segregated Ethereum wallet. Private keys are encrypted and stored securely. You can create additional wallets as needed.

Yield Accounts

Yield accounts hold your cryptocurrency deposits and earn 6% annual yield. TBD automatically routes funds to the best available protocol (Morpho or Aave).

Domain Guides

Accounts

Manage your developer account, update credentials, and view account details.

Key Operations:

  • Create account: POST /v1/accounts
  • Get account: GET /v1/accounts/{account_id}
  • Update account: PATCH /v1/accounts/{account_id}

Applications

Create and manage applications. Each application automatically gets an Ethereum wallet and can have multiple API keys.

Key Operations:

  • Create application: POST /v1/applications
  • List applications: GET /v1/applications
  • Get application: GET /v1/applications/{application_id}
  • Update application: PATCH /v1/applications/{application_id}
  • Create API key: POST /v1/applications/{application_id}/tokens

Wallets

Manage Ethereum wallets for your applications. Wallets are automatically created when you create an application.

Key Operations:

  • Create wallet: POST /v1/applications/{application_id}/wallets
  • List wallets: GET /v1/applications/{application_id}/wallets
  • Get wallet: GET /v1/applications/{application_id}/wallets/{wallet_id}

Yield Accounts

Create and manage yield accounts for earning interest on cryptocurrency deposits. Each account is backed by Morpho or Aave protocols.

Key Operations:

  • Create yield account: POST /v1/yield/accounts
  • List yield accounts: GET /v1/yield/accounts
  • Deposit funds: POST /v1/yield/accounts/{account_id}/deposit
  • Withdraw funds: POST /v1/yield/accounts/{account_id}/withdraw

Positions

View and manage your active yield positions across different protocols and cryptocurrencies.

Key Operations:

  • List positions: GET /v1/yield/positions
  • Get position: GET /v1/yield/positions/{position_id}

Markets

Explore available markets and protocols. TBD aggregates data from Morpho and Aave to show you all available yield opportunities.

Key Operations:

  • List markets: GET /v1/markets
  • Get market details: GET /v1/markets/{market_id}

Transactions

Track all transactions including deposits, withdrawals, and yield accruals across your accounts.

Key Operations:

  • List transactions: GET /v1/transactions
  • Get transaction: GET /v1/transactions/{transaction_id}

Webhooks

Subscribe to real-time events for yield accruals, position updates, and transaction notifications.

Key Operations:

  • Create webhook: POST /v1/webhooks
  • List webhooks: GET /v1/webhooks
  • Delete webhook: DELETE /v1/webhooks/{webhook_id}

Best Practices

Security & Compliance

API Key Management

Never commit API keys to version control. Use environment variables and rotate keys regularly.

Webhook Verification

Always verify webhook signatures using the webhook secret to prevent spoofing attacks.

Environment Separation

Use separate API keys for sandbox and production. Never mix test and live credentials.

RPC Security

Keep RPC API keys secure. Use environment-specific keys and monitor for unauthorized access.

Integration Patterns

Pattern 1: Simple Deposit Flow

Basic integration pattern for depositing funds and earning yield

// 1. Authenticate const token = await authenticate(username, password); // 2. Create yield account (if needed) const account = await createYieldAccount(token, { currency: "USDC" }); // 3. Deposit funds await deposit(token, account.id, { amount: "1000.00", currency: "USDC" }); // 4. Check position const position = await getPosition(token, account.id); console.log(`Earning ${position.yield_rate}% APY`);

Pattern 2: Webhook-Driven Updates

Event-driven architecture for real-time updates

// Your webhook endpoint POST /webhooks/tbd // TBD sends events: { "type": "yield.accrued", "account_id": "ya_...", "amount": "0.50", "currency": "USDC", "timestamp": "2025-01-15T10:30:00Z" } // Your handler: - Verify webhook signature - Update database - Notify users - Trigger additional actions
